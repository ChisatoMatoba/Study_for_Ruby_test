CSVアップロードガイドライン

このアプリではCSVファイルを使って問題を一括登録することができます。

CSVファイルの記載例

number question_content explanation choices
1 ソースコードの文字コードをUS-ASCIIに設定するマジックコメントの書き方として正しいものをすべて選択してください。 マジックコメントは「coding: エンコーディング名」が正しければ、その前後には任意の文字を並べることができます。また、大文字・小文字の区別もありません。 `# coding: us-ascii`~true,,`# encoding: us-ascii`~true,,`# -*- charset: us-ascii -*-`~false,,`# CODING: US-ASCII`~true
2 以下のコードを実行するとどうなりますか。
```
1:x = 0
2:def hoge
3: (1...5).each do |i|
4: x += 1
5: end
6: puts x
7:end
8:hoge
```		 メソッド内と外とでは、変数名が同じでも別の変数として扱われます。出題コードの1行目と4行目の変数xは、それぞれ別の変数です。hogeメソッド内の変数xは4行目が初出です。ここで自己代入演算子+=を使用して1を加算していますが、自己代入演算子は、最初に左オペランドを評価（値を参照）するため、定義されていない変数xを参照することになります。定義されていない変数を参照すると、例外が発生します。 `4`が表示される。~false,, `5`が表示される。~false,, 例外が発生する。~true,, `0`が表示される。~false

留意事項

  • ・CSVファイルはUTF-8で保存してください。
  • ・CSVファイルの拡張子は「.csv」にしてください。
  • ・CSVファイルの1行目はヘッダー行として、number, question_content, explanation, choicesの順で記載してください。
  • ・choicesは選択肢と正解の情報を「~」で区切って記載してください。正解の選択肢には「true」、不正解の選択肢には「false」を記載してください。
  • ・choicesの選択肢は「,,」で区切ってください。
  • ・question_contentとchoicesでは、markdown記法での記載が可能です。改行も反映されます。

CSVファイルのアップロード方法

  • ・csvファイルを選択し、アップロードして下さい。
  • ・同じ問題番号がある場合に、上書きするかどうかを選択できます。
